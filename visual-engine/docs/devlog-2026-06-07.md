# 開発ログ 2026-06-07（visual-engine / mixer 中心）

開発支援AIへの状況共有用サマリ。対象は `digital-cocktail/visual-engine`
（Three.js v0.184 + Vite + vanilla-ts）。1液ビューアーと2液混合ビューアー(mixer)を持つ。

---

## 0. 全体像（このリポジトリの2つのアプリ）

| アプリ | エントリ | 役割 |
|---|---|---|
| 1液ビューアー | `src/main.ts` | 単一カクテルの3D表示（既存・基準実装） |
| 2液 mixer | `src/mixer/mixer-main.ts` | 2液の色を設定→混合結果を3D表示 |

**設計の核：** mixer の結果ビューと小グラスプレビューは、1液ビューアーと
**「全く同じ3Dレシピ」**で描く。そのために材質・環境・後処理のファクトリを
`src/scene/cocktail-scene.ts` に集約し、両者が同一ソースを呼ぶ（単一ソース化）。

## 2. 機能別の詳細

### 2-1. 設定ビューの改良（`abaf077`）
- 設定タブが縦に伸びたらページ全体をスクロール可能に（共有 `style.css` が
  `html,body { overflow:hidden }` を掛けているため、mixer 側 `injectStyle()` で
  `html, body { height:auto; overflow:visible }` に上書き。タブバーは sticky で残す）。
- 「混合比率」「希釈」を**「混合パラメータ」折りたたみパネル**にまとめ、液体1の上に配置。
- スライダー直下に数値表示を追加：
  - 比率 → `液体1：XX%　液体2：YY%`
  - 希釈 → `希釈率；X.X倍`（式 `1/(1-w)`）
- 視認性向上のため、上記数値表示と各液体パネルの「濃さ ●% / 濁り ●%」を
  **赤系(`#ff7a7a`)**に。濃さ/濁りの赤字は `liquid-ui.ts` の readout に
  インラインspanで実装（ユーザー承諾のうえ1液ビューアーにも波及）。

### 2-2. 色混合エンジンの濁り補正（`abaf077`／`src/mixer/mix-color.ts`）
- **問題**：液体1を100%（ratio=0）にしても、色相差由来の濁りボーナスが乗ってしまい
  「液体1単体と完全一致しない」。
- **修正**：濁りボーナスを「実際に混ざった度合い」でスケール。
  `mixAmount = 4 * ratio * (1 - ratio)`（ratio=0→0 / 0.5→1 / 1→0）を乗算。
  → 片方100%では濁りボーナス=0。中央で最大。
- 併せて「2液系はそこまで濁らない」という体感に合わせ
  `TURBIDITY_BONUS` を 0.4 → **0.12** に低減。

### 2-3. 小グラスプレビューを1液と同一レシピ化（`eeb915b`）
共有ファクトリを `cocktail-scene.ts` から切り出し、`glass-view.ts` と
`buildCocktailScene` の両方で使用：
- `createEnvMap(renderer)` … 環境マップ(PMREM/RoomEnvironment)
- `createRichGlassMaterial()` … グラス材質（透過・IOR・減衰・クリアコート）
- `createRichLiquidMaterial(geo, useGradient)` … 液体材質＋縦グラデ(onBeforeCompile)
- `makeSetLiquidAppearance(mat)` … 色/濃さ/濁りの反映ロジック（クロージャ）
- `addPostProcessing(composer, scene, camera, w, h, {useDof, focus})` … bloom+DOF+Output

小グラス側の調整（1液との差分は「バー環境を入れない」「寸法に合わせた再照準」のみ）：
- グラス形状は**ショットグラス**（`SMALL_GLASS_STYLE = 1`）。
- ライト・カメラ・DOF合焦距離をショットグラス寸法に合わせて相対座標で再算出。
- **バー環境（机・バックバー写真・グラデ背景・接地影）は入れない**＝質感とライトのみ。
- **重要な制約**：EffectComposer(bloom/DOF/Output)を通すと**透過が失われ不透明出力**に
  なる。そのため真の透過は使えず、パネル枠色に合わせた単色背景
  (`BG_COLOR = 0x262630`、ACESトーンマップ後に本体背景 rgb(14,14,18) と一致するよう校正)
  を背後に敷いて溶け込ませている。

### 2-4. 小グラスのライト・カメラ調整（`eeb915b`, `e5fbfc8`, 以降の微修正）
- **横スポット廃止**：元はマティーニのステム視認用ギミック。小グラスには不要なので削除。
- **上ライトを面光源(RectAreaLight)化**：点光源スポットだと滑らかなグラス面に
  「点」が鏡映りして“細い縦スジ”の鏡面ハイライトになる。面積を持つ光源にして
  太く柔らかいハイライトに（`RectAreaLightUniformsLib.init()` が必要）。
  ハイライトは `TOP_LIGHT_X_OFFSET` で右へオフセット。
  ※鏡面の太さは本質的には表面roughnessと光源サイズで決まる、という整理を共有済み。
- **カメラ**：軽い見下ろし＋寄り。`glass-shapes.ts` の `buildSmallGlassShape` で
  `cameraPos: [0, cy + height*1.05, dist]`, `cameraTarget: [0, height*0.55, 0]`,
  `dist = height*2.0 + 1.0`。液面の楕円が少し見える初期アングル。
- **操作可に変更**：両小グラスを `interactive: true`（OrbitControls＋描画ループ）に。
  設定タブ表示中だけ `start()`、結果タブへ移動で `stop()`（負荷管理）。
  → ぐりぐり回せるので、ハイライト位置はユーザーが自分で変えられる。

### 2-5. 結果ビューの混合結果パネル調整（`0816c87`）
- **パネルがタブバーに重なる問題**：浮きパネルは `position:fixed; top:16px` 固定で
  mixer のタブバー(sticky・高さ約47px)に重なっていた。`liquid-ui.ts` に任意オプション
  **`fixedTop`**（既定 `'16px'`）を追加し、結果UIだけ `'64px'` を指定（1液は無変更）。
- **パネル開閉でグラスを左寄せ**：1液 `main.ts` の `applyPanelView` と
  **同一ロジック・同一値**を mixer の結果カメラ向けに複製（`applyResultPanelView`）。
  - スマホ時のみ作動（`IS_MOBILE = innerWidth <= 768`）。
  - 開いたら `camera.position = (0,5.5,7.0)`, `zoom = 0.9`,
    `setViewOffset` で画面幅の26%ぶん左へ寄せる。閉じたら復元。
  - 結果UIの `onToggle` に接続。リサイズ時・結果タブ復帰時にも再適用。

## 3. デプロイ / URL 構成（Vercel）

### 経緯
- 当初 `vercel.json` の `rewrites` で `/` → `/mixer.html` を試みたが**効かなかった**。
  → Vercel の `rewrites` は **afterFiles**（ファイルが実在しない時だけ発火）。
    `index.html` が実在する `/` では発火せず、1液のままだった。
- **最終方式**：ビルドの `index.html` 自体を mixer にする（リライト非依存）。
  - `index.html` … 2液 mixer（`/src/mixer/mixer-main.ts`）
  - `single.html` … 1液ビューアー（`/src/main.ts`、旧 index.html。中身=main.ts は無改変）
  - `mixer.html` と `vercel.json` は削除。
  - `vite.config.ts` の入力を `{ main: index.html, single: single.html }` に更新。

### 現在の公開URL（本番・確認済み）
| URL | 中身 |
|---|---|
| `https://digital-cocktail.vercel.app/` | 2液 mixer |
| `https://digital-cocktail.vercel.app/single.html` | 1液ビューアー |
| `https://digital-cocktail.vercel.app/mixer.html` | 廃止（404） |

GitHub と Vercel は連携済みで、`main` への push で自動デプロイ。

## 4. 主要ファイルと役割（mixer 関連）

- `src/scene/cocktail-scene.ts` … **単一ソース**。材質/環境/後処理のファクトリ＋
  `buildCocktailScene`（1液とmixer結果ビューが共用）。`EXPOSURE`, `PIXEL_RATIO` 等も export。
- `src/mixer/mixer-main.ts` … mixer のエントリ。タブUI、設定パネル、結果ビュー、
  小グラス2つの生成と負荷管理、`applyResultPanelView`。
- `src/mixer/glass-view.ts` … 小グラスの自己完結ビューア（共有ファクトリを使用）。
  上面光源(RectAreaLight)、`start/stop/renderOnce/resize`。
- `src/mixer/glass-shapes.ts` … グラス輪郭(LatheGeometry用)・カメラ・液体プロファイル生成。
- `src/mixer/mix-color.ts` … 2液混合の純粋関数（`mixTwoLiquids`, `hsvToRgb`）。Three.js非依存。
- `src/liquid-ui.ts` … 色設定パネル（1液/mixer設定/mixer結果で共用）。`fixedTop` 追加済み。

## 5. 既知の制約・注意点

- **EffectComposer は透過を不透明化する**。小グラスの背景は単色 `BG_COLOR` で代替。
  背景色は ACES トーンマップ後の見えで校正している（変更時は再校正が必要）。
- **検証手段**：`preview_screenshot` は WebGL ループでタイムアウトしがち。
  色の数値確認は eval + readPixels（2Dキャンバスへ drawImage）で実施。
  `location.reload()` を eval 内で呼ぶと「navigated or closed」エラー → reload と
  read は別 eval に分ける。
- **tsconfig**：`verbatimModuleSyntax`（`import type` 必須）、`erasableSyntaxOnly`
  （enum/namespace/パラメータプロパティ禁止、class は可）、未使用変数/引数エラー。
- 要望2の左寄せはスマホ幅(≤768px)のみ作動。PCの広い画面では発動しない。
