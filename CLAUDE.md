# デジタルカクテル（digital-cocktail）

## プロジェクト概要
カクテルの材料・分量・技法から「色」と「味」を計算するエンジン群。
最終的にはNext.js/Vercelのフロントに載せる。現在は純粋ロジックを
Vite + vanilla-ts 環境で開発中。

## リポジトリ構成（モノレポ）
digital-cocktail/
├── color-engine/   … 色エンジン（完成済み・本ドキュメント時点）
└── taste-engine/   … 味エンジン（未着手）
色と味はサブフォルダで分け、将来ひとつのカクテルシステムとして統合する。

## 色エンジンの設計（color-engine/）

### 処理フロー（4レイヤー・順序厳守）
1. 【前処理・物理】技法に応じた加水（水をitemsに追加）
2. 【化学】       加水後のitemsから暫定pH算出 → reactive材料のRGBをLUTで書き換え
3. 【光学】       ランベルト・ベールの法則で減法混色
4. 【後処理・物理】SHAKE時のみ 明度アップ＋opacityダウン
※物理レイヤーは前後2か所に分かれる。加水は化学の前、SHAKE補正は光学の後。
　加水を化学の前に置くのは、薄まったpHで変色を計算するため（より自然な挙動）。

### ファイル構成（color-engine/src/）
- engine/types.ts       … 材料(Ingredient)・レシピ(Recipe)の型定義
- engine/ingredients.ts … 材料マスタ（人が選んで入れる材料のみ。水は含めない）
- engine/optics.ts      … ランベルト・ベール減法混色
- engine/chemistry.ts   … pH算出（[H⁺]ベース）＋ pH→色LUT
- engine/physics.ts     … 加水(addWater) ＋ SHAKE補正(applyShakeFilter)
- engine/cocktail.ts    … 統合関数 mixCocktail（4レイヤーを順序通り実行）
- main.ts               … 検証ビュー（色を並べて目視確認。事実上のテストスイート）

### 主要な設計判断とその理由
- **言語/構成**：色エンジンはフロント非依存の純粋関数群。後でNext.jsへ移植する前提。
- **opacityの扱い**：吸光度の乗数（色素濃度の代理）として使用。クリアな材料(opacity低)は
  混色に効かず、濃い材料は強く効く。最終opacityは加水後itemsの体積加重平均で算出。
  ※注意：opacityが「色素濃度」と「見た目の透明度」の二役を担っている。将来問題になれば
    変数を分離する可能性あり。
- **pH計算は[H⁺]ベース加重平均**：単純なpH値の平均ではなく、各材料のpHを水素イオン濃度
  [H⁺]=10^(-pH) に変換して体積加重平均し、再びpHに戻す。理由：pHは対数スケールなので、
  単純平均だと現実的な分量の酸では変色が起きにくいため。化学的にも正確。
- **レモン果汁のpHは3.0**（実際の値2.3ではなく）：[H⁺]ベースだとレモン原液(pH2.3)は
  少量でも強く効きすぎ、青→紫の中間帯を飛ばして一気に赤へスナップしてしまう。
  「希釈済み果汁」相当のpH3.0に緩めることで、5/15/30mlで段階的な青→紫→赤を実現。
  ※これは色の演出のための調整値。味エンジンでpHを酸味に使う場合、真値(2.3)とは
    要求が相反するため、変数分離の検討が必要（下記「未決事項」参照）。
- **加水率**：BUILD/STIR=18%、SHAKE=22%（総容量比）。水はrgb=[255,255,255]・opacity=0・
  pH7.0で、色には寄与せずpHのみ薄める。physics.ts内で生成（マスタには入れない）。
- **SHAKE補正**：明度 各成分×1.12（255クランプ）、opacity×0.85。気泡の乱反射と濁りの疑似再現。

### LUT（pH→色、butterfly_pea）
pH 7.0→[30,50,200]青 / 5.0→[120,60,180]青紫 / 3.0→[200,50,110]マゼンタ / 1.5→[220,60,90]赤。
中間値は線形補間、範囲外は端にクランプ。

### 非採用ロジック（仕様書より）
クベルカ・ムンク理論、3DボリュームレンダリングはGPU負荷が高く費用対効果が低いため除外。
ウーゾ効果等の白濁はイベントフラグのif処理で疑似再現する方針（未実装）。

## 未決事項（味エンジン着手時に決める）
- **pHを味エンジンでも使うか**：色エンジンのpHは変色演出用に調整済み（レモン3.0）。
  味で酸味をpHで表現するなら真値(2.3)寄りが欲しく、要求が相反する。
  → 味エンジンが酸味を独自パラメータ(例:sourness)で持つか、pHを2役に分離するかを決める。
